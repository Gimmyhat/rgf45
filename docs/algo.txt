Преобразование шейп файла в XML (текущая версия)

Уникальность объекта (ключ) задается следующим набором атрибутов:
Субъект РФ + Название объекта

Субъект берется из поля шейпа SUBRF

Название объекта берется из полей шейпа:
заявки - FULL_NAME
НФ - NAME_GBZ
лицензии - SNOMV

Проверки:
- название шейп файла должно содержать слова 'заявки', 'НФ', 'Лиц'
- наличие поля Названия объекта
- наличие поля Субъекта
- субъект должен быть из справочника Росреестра
- повтор ключа (за исключением объектов заявок и лицензий из разных файлов - геометрии объединяются)
- проекция ГСК11
- валидность геометрии
